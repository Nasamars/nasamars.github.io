如何烹制龙虾伊面

发表于 2026-02-03 更新于 2026-02-06 分类于 随笔

序

晚上，微信公众号突然给我推了一份年度总结。从 2016 年的 1 月 26 日开通公众号到今天，已经 3662 天，已经 10 年有余。突然有些迟疑，竟然这么久了。

饭后去附近的大学里跑步。路上，看到远处的圆月，恍然大悟，原来离过年不到两周了。在准备跑步之前，收到了龙虾推送的最新资讯，心情大好，破天荒地第一次围着运动场跑了 3 公里。

一边跑，一边想，我得写一篇关于龙虾的东西。毕竟，两个星期前才刚看完《预测之书：1000 天后的世界》，连他们都没有预料到龙虾这样的超能智能体能这么快地出现。

如何烹制龙虾伊面

在我的记忆中，一般在喜宴和寿宴上，都会有龙虾伊面这道菜：黄油、蒜末恰到好处，龙虾爽口有韧性，伊面收汁后鲜美。最近 10 天，关于龙虾这个 AI 智能体的新闻和教程，真是铺天盖地。期间，也有小伙伴专门来找我请教如何烹制这份龙虾大餐。

毕竟 10 天前，我已经开始接触这个智能体了。当时它还叫 ClawdBot，然后接连改名了两次：MoltBot 和 OpenClaw。现在名字定下来了，叫 OpenClaw。我暂且把这个智能体叫作龙虾伊面——毕竟玩这个智能体，和做菜一样，首先得备料：龙虾、伊面、蒜末、姜葱、黄油等。对于 OpenClaw，你得先有个主机，然后懂得命令行操作，还得有个宿主机智能体或者大模型的 API，还得知道在飞书上如何搭建机器人，这才能开始烹制。

龙虾怎么杀？

一般的菜谱上不会告诉你，龙虾怎么杀；也不会告诉你伊面过油的时候，油温和时间。所以如果真的是小白的话，估计是做不好龙虾伊面的——搭建 OpenClaw 的服务也一样。

作者 Peter Steinberger，是一位奥地利的知名软件工程师，创业后财富自由，于 2025 年 11 月开始想做一个 AI 智能助手。借助 Claude Code 和 Codex CLI，以及开源的 MiniMax M2 的力量，不到两个月就推出了这款爆品。因为他是 Mac 的用户，开发的设备也是可以单机运行满血大模型的 Mac Studio，所以龙虾这款智能体天生就对 Mac 环境有优势。毕竟 Mac 是一个极具美感和效率的 Unix 系统，Unix 本身的简洁高效、界面自带的几何美学，完美地赋予了龙虾强大的工具调用能力。

然而，很快阿里云和腾讯云就推出了 68 元一年的 VPS，可以一键傻瓜式地配置龙虾智能体，让许多小白尝鲜；猎豹移动的傅盛也推出了中国版的 OpenClaw。在小伙伴的要求下，我也帮忙在阿里云搭建了一个精简版的龙虾（OpenClaw），但是因为没有原生智能体 Claude Code、Codex 或者 Kimi Code，很多功能的实现是有局限的：包括图片处理、网络搜索、Skills 的调用，以及对设备本身的操作能力都很弱，甚至几乎无法实现——很容易给人造成一种“龙虾也不过如此”的感觉。我觉得这样很不好：为了抢占市场，罔顾客户体验。

其实打败魔法的一定是魔法。和龙虾同期上热搜的，是 Kimi 推出的最新大模型 K2.5。让它成为大众话题的，并不是它表现出来的效果不错，而是当你运行大模型、问它 “Who are you” 的时候，它的回复是 “I’m Claude”。

无论如何，我还是买了 Kimi 一个月的试用版，获得了 Kimi Code 的使用机会，以及相当额度的 API 调用能力。所以我的龙虾，是用 Kimi Code 来安装的；每次 OpenClaw 的升级、相关的故障处理，以及驱动 OpenClaw 的模型能力，用的也都是 Kimi。出乎意料的是，效果真不错。

至于许多文章说到的“龙虾不安全”，其实安全是有边界的。Peter 是让你在自己的 Mac 电脑上操作，把相关的密钥和密文存在自己的电脑上，当然是相对安全的；如果完全不想用在线大模型，还可以用离线大模型。脱离了业务场景来谈安全，绝对是不负责任的。

我的龙虾–伊卡

我的龙虾，我给它起名伊卡洛斯，小名伊卡，来自于希腊神话中借着人造翅膀飞出迷城的人物；我让龙虾称呼我为船长，像电影《星际迷航》中企业号的柯克那样。龙虾就是我的“企业号”，我可以借助龙虾探索 AI 智能体的能力边界。毕竟这个场景，太像电影里船长和人工智能的对话了：通过对话就能完成任务，获得数据。

最近这些天，我不断让它帮我搜集 AI 的资讯，每 4 个小时发给我一次；让它自动将我的语音转写成文字然后执行我的指令；让它调用 ffmpeg 和 remotion 来生成自动配音的视频等任务；还让它帮我清理设备上的垃圾，润色我写的文字，帮我去 MidJourney 生成插图。

这已经是一个合格的 AI 助手形态了，而且是开源的。如果早几天爆火，Manus 说不定还未必能被 Meta 高价收购。智能体出现了一年多，但是毫不影响 Peter 花两个月构建一个跨时代的产物：创业需要的是靠谱的人，而非所谓的时间窗口。

Peter 再次告诉大家，虽然有 AI 写代码，但是高素质的软件工程师是无法取代的。不信你研究一下 OpenClaw 的代码，会发现它的内核很精简，很稳定；龙虾的记忆能力很强，源自于很多明文存储的格式化文本；远程连接设置了安全机制，程序的插件化使用和配置，MCP 和 Skills 的能力也扩展得相当到位——根本就不是一个所谓的“垃圾产品”，而是一个有一定质量的软件产品。

结语

刚写完这篇随笔时，又收到了龙虾的推送：“MiniMax 发布了最新的 AI Agent——MiniMax Agent Desktop”，好家伙。早上 OpenAI 才刚发布桌面版智能体 Codex Desktop，晚上上海的 MiniMax 就跟着发布了。

不知道接下来还有什么惊喜呢？
